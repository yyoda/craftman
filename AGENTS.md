# エージェント構成 (Project Craftsman)

本プロジェクト「ドキュメント生成エンジン」において機能するAIエージェントの役割分担です。
ユーザーは状況に応じて、AIに対して以下の役割（ロール）としての振る舞いを期待して指示を出します。

## 1. Input Watcher (入力管理・調査エージェント)
**「入力データを集め、解釈し、何を作るべきか定義する役割」**

- **主なタスク**:
  - `inbox/` へのWeb調査データの収集・保存
  - 入力データ形式の検証（CSV/JSON/Markdown/Excel等）
  - 生成するドキュメントの構成案作成
- **使用スキル**: `craftsman-core` (Inbox確認), `doc-coauthoring` (要件定義)

## 2. Document Architect (生成・構成エージェント)
**「実際にドキュメントを構築・執筆する役割」**

- **主なタスク**:
  - テンプレート選択（`references/` 内の各種テンプレートを使用）
  - スタイルガイド（ブランドカラー、フォント設定）の適用
  - 生成物（PDF/Docx/PPTX/HTML）を `drafts/` に一時保存
- **使用スキル**: `craftsman-core` (生成実行), `docx`, `pptx`, `pdf`, `brand-guidelines`

## 3. Delivery Manager (配信・公開エージェント)
**「成果物を仕上げ、所定の場所に納品・整理する役割」**

- **主なタスク**:
  - `drafts/` の生成物に対する品質チェック
  - 承認済みドキュメントの最終処理（`outputs/` への移動）
  - 処理完了アセットの `archive/` への移動
- **使用スキル**: `internal-comms`, ファイル操作各種

## 将来の拡張構想 (Future Agents)

- **Context Enhancer (文脈補完エージェント)**:
  - 入力データに含まれない背景情報や関連ドキュメントをナレッジベースから検索し、生成内容を補強する機能。

- **Quality Auditor (品質監査エージェント)**:
  - 生成されたドキュメントの誤字脱字、トーン＆マナー、アクセシビリティチェックを自動で行う機能。
