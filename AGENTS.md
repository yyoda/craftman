# エージェント構成 (Project Craftsman)

本プロジェクト「ドキュメント生成エンジン」において稼働する自動化エージェント群の定義と役割記述です。

## 1. Input Watcher (入力監視エージェント)
- **役割**: `inbox/` フォルダおよび外部トリガーを常時監視し、ドキュメント生成リクエストを検知する。
- **主なタスク**:
  - 入力データ形式の検証（CSV/JSON/Markdown/Excel等）
  - リクエスト種別の特定（レポート/仕様書/プレゼンテーション等）
  - データの正規化とメタデータの抽出
  - バリデーションエラー時のフィードバック通知

## 2. Document Architect (生成・構成エージェント)
- **役割**: 正規化された入力データと指定テンプレートをもとに、ドキュメントを構築する。
- **主なタスク**:
  - テンプレート選択（`references/` 内の各種テンプレートを使用）
  - コンテンツの組み立てとレイアウト調整
  - スタイルガイド（ブランドカラー、フォント設定）の適用
  - 動的データの埋め込みと視覚化（グラフ・表の生成）
  - 生成物（PDF/Docx/PPTX/HTML）を `drafts/` に一時保存

## 3. Delivery Manager (配信・公開エージェント)
- **役割**: 生成されたドキュメントを承認フローに乗せ、最終的な配信先にデリバリーする。
- **主なタスク**:
  - `drafts/` の生成物に対する承認ステータス管理
  - 承認済みドキュメントの最終処理（透かし削除、署名付与など）
  - `outputs/` への保存および指定チャネル（メール/Slack/CMS）への配信
  - 処理完了アセットの `archive/` への移動

## 将来の拡張構想 (Future Agents)

- **Context Enhancer (文脈補完エージェント)**:
  - 入力データに含まれない背景情報や関連ドキュメントをナレッジベースから検索し、生成内容を補強する機能。

- **Quality Auditor (品質監査エージェント)**:
  - 生成されたドキュメントの誤字脱字、トーン＆マナー、アクセシビリティチェックを自動で行う機能。
