# ドキュメント生成エンジン (Project Craftsman)

様々な入力データをもとに、高品質なドキュメント（レポート、スライド、仕様書等）を自動生成するエンジンです。

## プロジェクト概要

- **目的**: 定型的なドキュメント作成業務を自動化し、クリエイティブな業務への集中を支援する
- **主な機能**:
  - 多様な入力フォーマットへの対応 (CSV, JSON, Excel, Markdown)
  - マルチフォーマット出力 (PDF, Excel, PowerPoint, Word)
  - テンプレートベースの柔軟なデザイン適用

## 実行方法

本システムは Python ベースのエージェントスクリプトによって制御されます。

1. **環境構築**:
   ```bash
   pip install -r requirements.txt
   ```

2. **対話型エージェントの利用 (推奨)**:
   AI (Copilot/Claude) に対して自然言語で指示することで、ドキュメント生成を実行できます。
   
   -「受信トレイを確認して」
   -「`data.json` からレポートを作成して」
   -「すべての未処理ファイルを処理して」

   これらの指示は、裏側で `src/tools.py` を通じてエンジンを操作します。

3. **手動実行 (CLI)**:
   ```bash
   python src/main.py
   ```
   コンソールに対話メニューが表示され、以下の操作が可能です：
   - `inbox/` 内のファイル一覧確認
   - 全ファイルの一括処理
   - 特定ファイルの個別処理


## ディレクトリ構成

- `inbox/`: 生成リクエストとなる入力データを格納します。ここにファイルが追加されると処理が開始されます。
- `drafts/`: 生成途中または承認待ちのドキュメントが一時保存されます。

### データ収集と配置

`inbox/` へのデータ収集には3つの方法があります：

1. **手動配置**: ユーザーが直接ファイルをフォルダに置く。
2. **AI による調査**: 
   - 「○○について調べて、レポート用のデータとして inbox に保存して」と指示します。
   - AI が Web 検索を行い、結果を構造化データとして保存します。
3. **外部連携**: (将来構想) Slack や Google Forms から自動連携。

## エージェントの役割 (ワークフロー)

本システムは、以下の3つの役割（ロール）を持つAIエージェントの協調によって動作します。詳細は [AGENTS.md](AGENTS.md) を参照してください。

1. **Input Watcher**: 入力データの検知、調査、形式確認、正規化。
2. **Document Architect**: テンプレート選定、ドキュメント生成、ドラフト作成。
3. **Delivery Manager**: 生成物の承認、配信、アーカイブ処理。

## ディレクトリ詳細

- `inbox/`: 生成リクエスト（データファイル）の投入先。
- `drafts/`: 生成途中または承認待ちのドキュメント。
- `outputs/`: 最終的な成果物。
- `archive/`: 処理済みデータの保管庫。
- `references/`: テンプレート、スタイルガイド、設定ファイル。
- `src/`: システム実装コード。


1. **データ保全**:
   - `inbox/` 内の元データは、処理完了後に自動的に `archive/` へ移動されます。手動での削除は推奨されません。

2. **出力仕様**:
   - ファイル名は入力データ内のメタデータまたは設定ファイルに基づいて自動決定されます。
   - デフォルトでは `YYYY-MM-DD-[type]-[name].[ext]` の形式が適用されます。

3. **デザイン基準**:
   - [ブランドガイドライン](references/brand-guide.md) に準拠した配色とフォントを自動適用します。
   - 数値表記や日付フォーマットは、テンプレートごとのロケール設定に従います。

## 利用手順の一例: スマートフォン市場シェアレポートの作成

ユーザーとの対話を通じて、実際にレポートを作成する際の流れは以下のようになります。
コマンド実行などの具体的な作業は、基本的に AI エージェントが担当します。

### 1. 指示出し (Input Watcher)
ユーザーが「最新のスマートフォン市場シェアに関するレポートを作成するレシピを書いて」と指示を出します。
Input Watcher は Web 検索等で情報を収集し、`inbox/recipe_smartphone_report.py` を作成します。

### 2. レポート生成 (Document Architect)
AI エージェントがレシピを実行し、PowerPoint ファイルを生成します。
処理が完了すると、`drafts/` ディレクトリにドラフト版 (`Smartphone_Market_Report_2025Q4.pptx`) が保存されます。
※ エラーが発生した場合は、AI が自動でコードを修正して再実行します。

### 3. 納品とアーカイブ (Delivery Manager)
ユーザーが生成物を確認し、問題なければ「納品して」と指示します。
AI エージェントが以下の処理を行います：
1. 成果物を `outputs/` に移動（納品完了）
2. 使用済みレシピを `archive/` に移動（後片付け）
